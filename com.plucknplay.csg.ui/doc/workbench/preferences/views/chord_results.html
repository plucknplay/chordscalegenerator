Preferences: Chord Results View

You can set the following preferences for the Chord Results View:

Entries per Page

Here you can define the number of result entries per page.
Note: The less entries you allow per page the more pages will be needed.

Visible Columns

Since this tabular view has many columns the view tends to become very wide and confusing. Here you can specify which columns shall be shown. You just have to select the columns you want to show. The first column "Chord" will always be shown and can not be hidden, since this column defines the chord and is therefore absolutely necessary.

Hide Columns if not needed

Here you can additionally specify whether the column "Without Root", "Without 3rd" or "Without 5th" shall be shown permanently or shall only be shown if it is needed to define at least on chord result.
